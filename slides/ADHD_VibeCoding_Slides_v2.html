brick 2 / 收敛

Skills + Coding Plan

省钱，想改几轮改几轮，但 Hyperfocus 改 3 小时停不下来

❌ 问题

每次写 Prompt 从头来

改着改着偏离主线

想优化 A 又冒出 B、C

✅ 解法

Skills 打包：固化常用模板，一键调用

Coding Plan：硬规则，不是建议

停止条件：输出 3 条即结束

关键洞察：ADHD 的大脑对 "建议" 会自动忽略，对 "规则" 会执行。

规则，不是建议——这句话很重要。我试过告诉自己"差不多就行了"，没用。但如果说"系统规定最多3条"，我就会停。
skill demo

课件复盘 Skill

【系统角色】 你是一位国际学校课程复盘助手 【核心规则 - 必须遵守】 1. 输出必须收敛：只给结论，不问问题 2. 输出必须有限：严格按指定数量 3. 输出必须结构化：固定格式 【处理流程】 1. 提取核心教学目标（严格限 3 条） 2. 列出学生常见误区（严格限 2 条） 3. 给出建议调整的 3 个点 【禁止行为】 ❌ 不提供替代方案 ❌ 不生成补充材料 ❌ 不询问 "您是否需要..." 【结束语】 "复盘完成。如需深入某个点，请明确指定编号。"

预制菜 一键调用 不写 Prompt

这就是我说的"预制菜"——不用每次重新炒，随时热一下就能用。但光有工具还不够，我要能管住自己的念头。

the trap

ChatGPT 的"要 A 还是 B？"

对 ADHD 是灾难，不是体贴

ChatGPT 说 ADHD 内心 OS
"您想要正式还是随意的语气？" "正式吧...但随意可能更自然...等等，看对象是谁..."
"需要我展开 A 还是 B？" "A 吧...但 B 也有道理...要不都看看？...C 也不错？"
"您还有其他需求吗？" "有！但我不确定是什么...让我想想...2 小时过去了"

💡 我的收敛开场白

"不要问我问题，直接给出你的最佳建议，我会反馈调整。"

你看——ChatGPT 以为在帮忙，其实是在挖坑。它每问一个问题，我就得多想一个半小时。
