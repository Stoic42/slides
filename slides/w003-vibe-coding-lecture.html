开场破冰的核心目标是建立安全感，打破"我不懂技术"的恐惧，完成 Tech Lead 与 Creator 的混编。

"欢迎来到 She Code Lab W003。今天在座的没有'程序员'和'小白'之分，只有'想做产品的人'。"

"我们要抛弃传统的编程学习路径（先学半年语法）。今天，英语（或中文）就是最高级的编程语言。"

"只要你能把你的想法说清楚，你就能写代码。"

"Campfire" 座次安排策略

16 位 Tech Lead 分散坐开，Creator 坐在附近——这样 Creator 可以 solo 工作，但遇到问题可以转身问 Tech Lead。

Tech Lead 识别（5分钟）

  • "请收到'Builder'邀请的 Tech Lead 站起来或举手。"
  • "这些是你们的'救生员'。如果环境配置出问题，或者 GitHub 拒绝你的密码，找她们。"

"Hello World" 环境检查（15分钟）

投屏显示三个绿灯检查：

  • ✅ 工具：打开 Cursor/Trae，确保 AI Chat 能用
  • ✅ 云端：登录 GitHub Desktop，创建空白 repo "W003-Demo"
  • ✅ 链接：（可选）登录 Vercel，连接 GitHub 账号

规则："如果你有 3 个绿灯，可以去接咖啡。如果有红灯，问最近的 Tech Lead。"

二、工具链底座  00:30–01:00

工具选型指南

全自动 Agent · 适合从 0 到 1

Trae

还没想好代码怎么写，完全生成模式。Trae 帮你写 PRD、建文件、写代码。

PM 思维可视化 · 适合逻辑复杂

Kiro

逻辑复杂、需要先写文档。先 Design 再 Task，把 PM 思维可视化。

局部修改 · 适合已有代码

Cursor

已有代码，局部修改。最强补全 (Tab)，反应快。

网络分流策略

情况推荐模型
可以翻墙Claude 3.5 Sonnet / GPT-4
不能翻墙DeepSeek / Kimi / Trae 国内版

重新理解 API Key：麦当劳得来速 + 贴牌比喻

API Key = 企业信用卡：让你能访问大模型工厂（DeepSeek/Qwen 等）

Prompt = OEM 贴牌生产单：决定产出是否符合你的品牌调性

安全警告：不要把 API Key 写在公开代码里，用 .env 文件

救命锦囊（直接挂屏幕上）

  • Terminal 密码输入不显示："输入 sudo 密码时屏幕不会有反应，这是正常的！输完回车就行。"
  • Git 弹窗轰炸："Mac 疯狂弹 Git 验证？点取消，或者切回 IDE 关掉 Git 插件，今天用 GitHub Desktop 拖拽上传。"
  • 依赖报错："看见 command not found 或红字？把报错信息复制，扔回给 AI，让它教你装环境。"
  • 永远先建文件夹：不要把项目生在根目录！

三、核心方法论：Prompt 秘籍  01:00–01:40

Prompt 公式

Context（背景） + Task（任务） + Constraints（约束） = Magic ✨

要素含义举例
Context 背景信息，把图片拖进去，把日记粘贴 "照着这个图做"、"这是我写的第一篇文章"（附文章内容）
Task 具体任务，说明白目标和结果 "做一个红色圆形按钮，点击弹出'Hello'"
Constraints 约束条件，格式、风格、边距……说清楚 "用 Bootstrap，响应式，不要动画"

迭代思维：AI 第一版永远是不完美的

如何通过"追问"来修改 Bug：把报错信息复制给 AI，问它"这是什么意思，请帮我修好"。这就是程序员每天 80% 的工作。

现场演示：故意出错，然后修复

亲自演示用 5 分钟从零写出一个简单的"倒计时番茄钟"——演示出错的情况，然后演示如何把报错信息复制给 AI 让它修好。

五、痛点验票  02:00–02:30

便利贴法则

写

Step 1: Write

拿出便利贴：

  • 左边：你的 Idea（产品名）
  • 右边：崩溃瞬间（即你的 Context）——你生活中最烦恼的那个具体场景
验

Step 2: Check

自检时刻：

  • 写不出右边的"崩溃瞬间"？
  • 警报：你的 Idea 太大了！
  • 立刻缩小范围，直到你能写出那个瞬间。

你的痛点 = 最好的 Prompt 输入

AI 不仅是你的程序员，它首先是你的镜子。今天我们不一定要做一个改变世界的产品，但我们可以做一个"懂我"的产品。

六、上线部署  02:30–03:00

分流部署

情况推荐方案
可以翻墙Vercel（推荐）：拖拽上传或连接 GitHub
不能翻墙EdgeOne Pages（腾讯云）：国内可用，支持全栈

万能部署 Prompt

"我是新手，我想把当前的网页免费部署上线。我已经安装了 Vercel 插件（或者请教我用最简单的方式）。请不要让我用复杂的命令行，最好能用图形界面或者最简单的指令。"

保底大招：Netlify Drop（把文件夹直接拖进网页即可上线，无需 Git）

关于"刹不住车"：分流策略

不要试图让跑得快的人停下来听你讲基础，这会消磨她们的热情。

动作：开场直接给一个"Cheat Sheet / 路线图"（一张 PPT 或 Notion 链接）。话术："如果你已经知道要做什么了，并且环境是绿灯，请直接开始 Solo，不用听我讲废话。如果你卡住了，或者不知道选哪个模式，请抬头看屏幕。"

七、Demo Day  03:00–03:30

Pitch 规则

闪电演讲，每人 2 分钟：

  1. 我原本想做什么？
  2. 我遇到了什么困难？
  3. AI 是怎么帮我解决的？
  4. 展示那个 URL。

代码躺在你的电脑里只是文件，只有 URL，才是产品。

AI 帮你吹牛（下午 4 点强制执行）

打开 Gamma，输入："我做了一个 [产品一句话介绍]，解决了 [什么痛点]，核心功能是 [1,2,3]，请帮我生成 5 页 Pitch Deck。"

或者让 Trae/Cursor 生成一个漂亮的 README.md，直接演示 GitHub Pages。

闪电演讲，每人 2 分钟，分享内容：

  1. 我原本想做什么？
  2. 我遇到了什么困难？
  3. AI 是怎么帮我解决的？

给 Tech Lead 的话

"试着挑战一下，不要写一行代码，纯粹用自然语言让 AI 完成你的架构。感受一下这种'指挥官'的感觉。"
