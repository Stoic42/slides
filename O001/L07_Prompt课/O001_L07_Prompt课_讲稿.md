### 10:00-15:00 为什么 AI 经常"听不懂"

我先问大家一个问题：

> 你跟一个实习生说"帮我做个登录页"——你觉得他会做出什么？

99% 的情况是：他会做出一个**最普通的登录页**。

为什么？因为你没告诉他：
- 用户是谁
- 什么场景
- 什么风格
- 什么约束

**AI 也是一样**。

> 99% 的"AI 不好用"都是你的问题。

AI 不会猜你想要什么。它会给你**最常见、最平庸**的版本。

```
❌ "帮我做个登录页"

✅ "我想要一个 22-28 岁职场女性使用的 App 登录页，风格温柔但专业，
   颜色用浅粉 + 白色，输入框圆角，按钮用品牌色，
   要显示 Apple/微信登录"
```

这两个 Prompt 的差距，就是**你和"AI 帮你做出好产品"之间的距离**。

> Prompt 不是打字比赛，是**认知整理**。

### 15:00-22:00 完整 Prompt 的四要素

**（PPT S03）**

一个完整的 Prompt 应该包含**四要素**：

| 要素 | 问自己 | 例子 |
|------|--------|------|
| **角色** | 让 AI 扮演谁 | "你是资深 UI 设计师" |
| **场景** | 给谁用、什么场景 | "22-28 岁的职场女性第一次打开" |
| **任务** | 具体要做什么 | "设计登录页，写 HTML + CSS" |
| **约束** | 必须满足的条件 | "浅色调，圆角，适配移动端" |

为什么这四个要素这么重要？

- **角色** 决定 AI 的"专业水准"和"思考角度"
- **场景** 决定 AI 的"目标用户"和"使用情境"
- **任务** 决定 AI 的"具体输出"
- **约束** 决定 AI 的"边界条件"和"红线"

> **PRD 写不清楚，灵光就生成不好。这不是 AI 的问题，是你的问题——你想清楚了吗？**

这是 M001 Vivan 的金句，也是我们 L12 商业化课的核心观点。

### 22:00-28:00 模板与实例

**（PPT S04）**

给你们一个**万能模板**：

```
你是 [角色]。

[场景描述]：[用户是谁，他们在什么情况下使用你的产品]。

[任务描述]：请帮我 [具体做什么]。

[约束条件]：
- 风格：[参考例子或描述]
- 技术：[用什么框架/语言]
- 限制：[必须做/不能做]
- 输出：[代码/文档/列表/...]

[验收标准]：[怎么判断做得好]
```

**现场演示**：用模板写一个灵光的闪应用 prompt。

```
你是一位资深的产品经理和前端工程师。

场景：22-28 岁的职场女性，她们经常出差或轮班，
需要快速计算不同起床时间对应的最佳入睡时间。

任务：帮我做一个睡眠计算器，能根据起床时间推荐
入睡时间，显示 R90 睡眠周期，并允许调整周期数。

约束：
- 风格：简洁优雅，主色调浅粉 + 白色
- 限制：不能调用任何外部 API（离线使用）
- 输出：可交互的网页

验收标准：
- 用户能选择 4-10 个不同的起床时间
- 自动计算对应的最佳入睡时间
- 直观显示睡眠周期
- 加载时间 < 2 秒
```

> 看，这就是 Vibe Coding 的"正确姿势"——用模板 + 真实场景 + 具体约束。
> 你不需要懂代码，但需要**想清楚要什么**。

### 28:00-34:00 进阶技巧

**（PPT S05）**

光有四要素还不够。下面是**四个进阶技巧**：

#### 1. 给 AI 看反例

```
我之前让另一个 AI 做的版本长这样：[粘贴]
我觉得太丑了，因为 [原因]。
请做一个明显更好的版本。
```

#### 2. 让 AI 自我评估

```
请先列出 3 个可能的方案，
然后说明每个方案的优缺点，
最后选一个最合适的，解释为什么。
```

#### 3. 链式 Prompt（多轮对话）

不要一上来就要"完美"。分轮迭代：

```
第 1 轮："先给我一个简单版本"
第 2 轮："现在加上暗色模式"
第 3 轮："把按钮颜色调成品牌色 #ff71ce"
第 4 轮："再加一个加载动画"
```

#### 4. 角色压力测试

```
你是一个挑剔的设计师，专门找 UI 的问题。
请审视我刚才给你的设计，列出 5 个最严重的问题。
```

这四个技巧的共同点：**让 AI 不偷懒、不敷衍**。

#### 5. 追问三技巧（AI 第一版答案很少是最佳）

**AI 的第一版回答很少让你满意**——这是正常的。关键是**怎么追问**让它越改越好：

**技巧 1：聚焦细节**（❌ 模糊 → ✅ 具体）

```
❌ "再改改"

✅ "把第三步的按钮改成圆角、字号加大到14px"
```

**技巧 2：多方案对比**（❌ 让 AI 猜 → ✅ 让 AI 发散）

```
❌ "换个方案看看"

✅ "给出 3 个不同风格的首页方案，解释各自适用场景"
```

**技巧 3：换视角追问**（❌ 让 AI 自评 → ✅ 引入新视角）

```
❌ "这样对吗"

✅ "如果用户是 60 岁视力不好的女性，哪里需要改？"
```

> 关键：**不要重复问同样的问题**——换"角度"问，不要换"力气"。

### 34:00-40:00 避坑清单

**（PPT S06-S07）**

最后是**避坑清单**：

| 坑 | 怎么避 |
|----|--------|
| Prompt 太短 | 至少包含四要素（角色/场景/任务/约束）|
| 一上来就要"完美" | 先 80 分，再迭代 |
| 不知道哪里错了 | 让 AI 复述它理解的需求，看跟你想的是不是一致 |
| 重复问同样的问题 | 把好的 Prompt 存起来，下次直接用 |

**课后行动**：

1. **必做**：用四要素模板写一个 Prompt，解决你生活/工作中的一个具体问题
2. **选做**：搭建你的个人 Prompt 库（Notion/飞书/NotebookLM 都可以）
