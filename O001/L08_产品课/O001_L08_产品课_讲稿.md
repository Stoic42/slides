### 8:00-15:00 痛点 / 痒点 / 爽点

**（PPT S03）**

今天我们用**梁宁《产品思维 30 讲》**的框架：痛点 / 痒点 / 爽点。

| 类型 | 定义 | 用户心理 | 例子 |
|------|------|----------|------|
| **痛点** | 必须解决，否则难受 | "我不解决就过不去" | 失眠 / 工资低 / 找不到工作 |
| **痒点** | 解决了更好，不解决也行 | "我想要但不是非必须" | 界面好看 / 多一个滤镜 |
| **爽点** | 解决了会上瘾 | "哇这也太爽了" | 一键生成 / 满减优惠弹窗 |

**核心原则：先做痛点，再做痒点，最后爽点。**

为什么？

- 痛点 = 用户**离不开**你
- 痒点 = 用户**更喜欢**你
- 爽点 = 用户**主动传播**你

正确的顺序是：**痛点立命 → 痒点增色 → 爽点裂变**。

### 15:00-22:00 真实场景对照

**（PPT S04）**

回到 sisi 的"活一下"：

| 满足 | 类型 |
|------|------|
| 职业倦怠的人有具体任务可做 | **痛点** ✓ |
| 完成任务有可视化反馈（种子成长）| **爽点** ✓ |
| 界面设计很温馨 | **痒点** ✓ |

> 看，痛点 + 痒点 + 爽点**全都有**。这就是好产品的结构。

**反例**：

- "我想做一个让用户分享生活感悟的 App" → 痒点？痛点？说不清 → **别做**
- "我想做一个让用户能 AI 修图" → 痒点（淘宝美图秀秀已经做过了）→ **红海**
- "我想做一个让年轻人 24 小时不下床" → 痛点不够真实（夸张）→ **别做**

> **99% 的初创死在"做出来"和"卖出去"之间**——M001 赵迪的金句。
> 而"卖不出去"的原因，80% 是痛点没找对。

### 22:00-32:00 怎么找到真正的痛点

**（PPT S05）**

给你三个方法：

#### 方法 1：问"你上一次……是什么时候"

不问"你想要什么"（人都想要好东西），
问**"你上一次遇到这个问题是什么时候，怎么解决的"**。

> 真实案例：你问女性"你想要更好的经期追踪 App 吗？"
> 她们说"想"——这是**痒点**。
> 你问"你上一次经期混乱是什么时候，怎么应对的"——
> 她们会**讲出真实故事**——这才是**痛点**。

#### 方法 2：找"忍了很久的事"

**最好的痛点 = 大家默默忍受很久的事**。

- 通勤时间长
- 外卖选择困难
- 跟同事沟通不顺
- 找不到好吃的餐厅
- 跟父母沟通难

> 默默忍受 = 没有现成解决方案 = 真痛点。

#### 方法 3：看自己

**你最痛的产品问题，往往是你自己的产品**。

- 失眠 → 做睡眠 App
- 找不到好吃的 → 做探店 App
- 跟父母沟通难 → 做家庭沟通工具
- 职业倦怠 → 做"活一下"

> sisi 做出了"活一下"——因为她自己就在经历职业倦怠。

**idea 验证三问**：

| # | 问题 | 答不上来 = 别做 |
|---|------|----------------|
| 1 | 谁会真的为这个付钱？ | 没有人 → 别做 |
| 2 | 他们现在用什么代替？ | 没有代替品 → 痛点不够真实 |
| 3 | 为什么是我能做？ | 别人也能 → 没壁垒 |

**痛点验证三步法**（@小苋 实操方法）：

> 别只问"你想要这个吗"——人都会说"想"。要靠真实验证。

1. **写出假设**：「我认为[某人群]有[某问题]，他们愿意为这个付钱」
2. **找 3 个真人聊**：**不能是朋友/同事**——他们会客套。去目标用户常出现的地方（线下活动/小红书/微信群），**问开放式问题**："你上一次遇到这个问题是什么时候？"
3. **看付费意愿**：3 个人都描述了真实场景 + 至少 1 个人说"愿意为这个付钱" → 真痛点。否则回去重写假设

> 关键：**没人付过钱 = 待验证**，不是"真痛点"。

## 附录 A：痛点 / 痒点 / 爽点的心理学基础

- **痛点**：马斯洛需求层次**底层**——生理/安全需求
- **痒点**：马斯洛需求层次**中层**——社交/尊重需求
- **爽点**：多巴胺奖励回路——即时满足/上瘾机制

## 附录 B：3 个访谈问题模板

```
1. 自我介绍：你是做什么的？日常生活是怎样的？
2. 痛点探索：上一次遇到 [X 问题] 是什么时候？
   - 你当时是怎么解决的？
   - 解决得怎么样？还差什么？
3. 替代方案：你现在用什么工具/方法应对 [X 问题]？
   - 它哪里好？哪里不好？
   - 如果有一个更好的工具，你愿意付多少钱？
```

## 附录 C：好产品的结构

```
痛点（用户离不开你）     ←── 核心
  +
痒点（用户更喜欢）       ←── 差异化
  +
爽点（用户主动传播）     ←── 增长
  =
好产品
```
