### 10:00-15:00 什么是 Vibe Coding

**（PPT S02-S03）**

你有没有过这种时刻：
- 在路上突然想到一个 App 创意
- 但打开编程教程看 5 分钟就放弃了
- 想着"如果有人能帮我实现就好了"

**Vibe Coding 是什么？**

Vibe Coding 是 Andrej Karpathy 在 2025 年提出的概念：
> *"你完全跟着感觉走，不看代码，只看结果。让 AI 帮你实现。"*

```
传统编程：   写代码 → 跑 → 报错 → 改 → 跑 → 部署
Vibe Coding： "我要一个这样的页面" → AI 生成 → 微调 → 发布
```

**关键区别**：
- 你不用懂代码，但你得**想清楚要什么**
- AI 是你的实习生，**你**是导演
- 关键决策还是你做

### 15:00-22:00 30 秒闪应用现场演示 ⭐

**（PPT S04）**

现在我要演示一个真实场景。

（打开灵光 APP 或 `lingguang.com`）

> **不要贪吃蛇**——之前的 workshop 用过，太无聊了。我们做你真正想用的产品。

> 我要做一个情绪日记首页。让我用一句话告诉灵光：

```
你是一位资深 UI 设计师。

场景：22-28 岁的职场女性，她们想要一个简单的方式记录今天的心情。

任务：帮我做一个情绪日记首页，让用户可以选择今天的心情
（开心/平静/难过/生气），点击后看到一句温柔的反馈。

约束：
- 风格：粉色系 + 白色，圆角按钮
- 限制：只能用一个主屏幕，单手操作
- 输出：可交互的网页，移动端优先
- 底部 3 个导航入口：日记 / 趋势 / 我的
```

（按回车，等待 30 秒）

（看到生成的应用：心情选择卡片 + 温柔的反馈 + 底部 3 个入口）

> 看，这就是一个**真实可用的应用**——我不用写一行代码。
> 4 要素（角色/场景/任务/约束）让 AI 一次就做出我要的，而不是平庸的"默认版本"。

现在我让它换个风格：

```
改成《黑客帝国》风格，黑色背景绿色代码瀑布流。
```

（几秒钟后界面变了）

> 这就是 Vibe Coding 的**迭代能力**——你想改什么，用自然语言告诉 AI 就行。

> 而且这个应用**可以分享**——我能保存到"我的闪应用"，
> 然后一键分享到微信、支付宝、灵光圈。

**这就是 Wish Coding 范式**——从"写代码"到"描述意图"。

### 22:00-28:00 AI = 读完了世界上所有书的实习生

**（PPT S05）**

很多人以为 AI 是万能的，但它不是。

把它想象成**一个读完了世界上所有书的实习生**：
- 他很博学
- 但他**没有真正的判断力**
- 他会犯错
- 你必须检查他的工作

| AI 能做 | AI 不能做 |
|--------|----------|
| 写标准代码 | 理解你的真实需求 |
| 解释概念 | 替你做价值判断 |
| 生成界面骨架 | 处理边缘情况 |
| Debug 报错 | 替代人类共情 |
| 翻译语言 | 理解文化差异 |

**怎么跟它合作？**
- 你是**导演**，AI 是**实习生**
- 你给方向，它执行
- 关键决策还是你做

### 28:00-34:00 70% 陷阱

**（PPT S06）**

有一个重要的概念叫 **70% 陷阱**：

> AI 擅长处理"**偶然复杂性**"——能在 5 分钟内完成标准网页代码和常规前端组件骨架。
> 但它无法处理"**本质复杂性**"——边缘情况、复杂系统架构、道德判断。

| 部分 | 谁负责 |
|------|--------|
| 写标准网页代码 | AI ✅ |
| 生成 UI 组件 | AI ✅ |
| 写 CRUD 逻辑 | AI ✅ |
| 边缘情况处理 | 人类 ⚠️ |
| 复杂系统架构 | 人类 ⚠️ |
| 物理概念 | 人类 ⚠️ |
| 道德判断 | 人类 ⚠️ |
| 错误信息的人性化 | 人类 ⚠️ |

**记住：AI 是工具，不是替代品。**

这意味着：
- **不要** 等 AI 写出"完美"代码 → 80 分就上线，让用户告诉你差在哪
- **不要** 让 AI 替你做价值判断 → 这是你作为设计师/产品人的责任
- **要** 把 AI 当成加速器 → 你专注在最关键的 30% 上
