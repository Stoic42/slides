### Slide 2：课前问题

### Slide 3：什么是边缘案例
```
边缘案例：不在"大多数"范围内的用户需求

举例：
健身 App → 大多数：能运动的人
          边缘：坐轮椅的用户

外卖 App → 大多数：能接电话的人
          边缘：听障骑手

社交 App → 大多数：能打字的人
          边缘：失语症患者
```

### Slide 4：真实案例
```
Apple Health（2014年）

追踪：睡眠、心率、营养、血压……
缺失：月经追踪

原因：设计团队以男性为主
      没有意识到月经追踪的重要性

→ 科技行业"设计多样性不足"经典案例
```

### Slide 5：怎么主动避免
```
问 AI："这个设计考虑了哪些边缘用户？"

边缘案例追问清单：
1. 老年人/青少年/非技术背景用户？
2. 视障/听障/行动不便/认知障碍？
3. 方言/少数民族语言/低识字率？
4. 低收入/弱网络/老旧设备？
5. 焦虑症/抑郁症用户？
```
