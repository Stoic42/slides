# C09 Pitch课讲义

> **学员手持材料**

---

## 一、Pitch 为什么重要

黑客松评委平均只看 5 分钟。

在这 5 分钟里，他们要看 10-20 个项目。

**如果你不能在第 1 分钟内让他们说"这个有意思"，他们就会把注意力转到下一个。**

---

## 二、Pitch 结构：30 + 20 + 10

| 部分 | 时间 | 内容 |
|------|------|------|
| **作品** | 30秒 | 这是什么 + 为谁做 + 解决什么痛点 |
| **亮点** | 20秒 | 最满意的一点 + 数据或具体场景 |
| **下一步** | 10秒 | 现在在哪 + 下一步 + 怎么联系 |

---

## 三、案例对照

### 好的 pitch：「活一下」App

> "我做了一款叫'活一下'的 App，专门帮助因为职业倦怠而丧失行动力的年轻人。它解决的是心理健康问题，但不像传统心理咨询那样需要预约和付费——用户只需要每天完成一个 1 分钟的盲盒任务。我已经在 App Store 上架，评分 4.8，下一步想融资继续增长。"

### 差的 pitch：功能列表型

> "我们的 App 有登录注册、个人中心、任务系统、成就徽章、数据分析报告、社区论坛、分享功能……"

### 差的 pitch：技术细节型

> "我们用了 React Native 开发，后端是 Node.js，数据库用的 MongoDB，部署在阿里云上……"

## 四、写出你的 Pitch

用 3 句话写出你的 1 分钟版本：

**第1句（作品）：**
我的产品是______，为______（用户）设计，解决______（痛点）。

**第2句（亮点）：**
我最满意的是______，数据显示______。

**第3句（下一步）：**
现在______，下一步______，联系我______。

---

## 五、Pitch 自查清单

| 问题 | ✓/✗ |
|------|------|
| 我在 30 秒内说清楚了"产品+用户+痛点"吗？ | |
| 我有给出一个具体的数据或场景吗？ | |
| 我有说清楚下一步（链接/融资/招募）吗？ | |
| 我避免了"功能列表型"pitch 吗？ | |
| 我避免了"技术细节型"pitch 吗？ | |
