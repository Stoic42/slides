```
在于 pitch 结构
```

### Slide 3：Pitch 结构
```
30秒：展示作品
→ 这是什么 + 为谁做 + 解决什么痛点

20秒：创作亮点
→ 最满意的一点 + 数据或具体场景

10秒：下一步
→ 现在在哪 + 下一步 + 怎么联系
```

### Slide 4：时间分配示例
```
「活一下」App pitch 示例

[0:00-0:30]
"我做了一款叫'活一下'的 App，
专门帮助因为职业倦怠而丧失行动力的年轻人"

[0:30-0:50]
"我最满意的是'盲盒行动'功能，
用户平均每天完成3.2个任务，
主动求助心理咨询的比例下降了40%"

[0:50-1:00]
"App 已上架，下一步想融资 100 万"
```

### Slide 5：常见错误
```
❌ 功能列表型
"我们的 App 有登录注册、个人中心、
任务系统、成就徽章、数据报告……"

❌ 技术细节型
"我们用了 React Native + Node.js + MongoDB"

✅ 价值聚焦型
"这是一个帮助 X 用户解决 Y 问题的产品"
```
