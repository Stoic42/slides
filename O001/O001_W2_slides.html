Part 0 AI助手 vs AI Agent

🤝 AI助手

被动响应 · 你问，它答

像餐厅服务员：你点菜，他上菜。
不推动任何事情，只响应你的输入。

→ 当前课程教的是这个

🏠 AI Agent

主动规划 · 你设目标，它自己干

像私人管家：你说"安排今晚约会"，它自动订餐厅、叫车、提醒你出发。

→ W3 Agentic Thinking会涉及

Part 1 双钻模型：先发现正确的问题

Don Norman《设计心理学》核心

设计第一步是"解决正确的问题"，不是急着给答案。

💎 第一颗钻石

发散→收敛

深入探索问题的根本原因（Root Cause）
不要跳到第一个显而易见的答案

💎 第二颗钻石

发散→收敛

找到产品的解决方案
基于第一颗钻石的真实洞察

练习：在动手写PRD前，先用5分钟写下"我真正要解决的是什么"

Part 2 痛点 vs 痒点

💔 痛点 Pain Point

用户真正愿意花钱或花时间解决的麻烦

  • 不做会难受
  • 做了会爽
  • 痛点驱动真实需求

✨ 痒点 Itch Point

用户觉得"挺好"但不是"必须"的功能

  • 做不做都行
  • 做了加分
  • 不做也不会离开
  • 适合做差异化，但不该作为MVP核心

MVP阶段：聚焦痛点，痒点靠边站。每次想加功能时问"删掉会怎样？"

Part 3 PRD五维度 + Garrett五层模型

PRD 五维度

  • 背景：为什么做这个
  • 用户：为谁做（用户画像）
  • 功能：做什么
  • 用户流程：怎么做
  • 数据需求：需要什么数据

Garrett五层模型（入门两层）

  • 战略层 Strategy：产品目标与用户需求——先回答"为什么做"
  • 范围层 Scope：功能与内容规格——明确"做什么"和不做什么

关键：砍掉"现在不做什么"和定义"做什么"同样重要

PRD = 给AI的"食谱" — 有食谱，AI才能做出你要的菜，而不是随便炒一盘

Part 3 用户画像 User Persona

用户画像 = 你为目标用户建立的虚拟形象

他们是谁、做什么、多大年纪、有什么痛点。
PRD里写清楚"为谁做"，AI才能生成对的语气和功能。

📋 必做：两个具体Persona（AI生成或手动写）

  • 基本信息：年龄、技术水平、职业
  • 痛点：这个产品解决他们什么问题
  • 使用场景：什么时候会用这个产品
  • 障碍：他们有什么顾虑或困难

Part 4 用户访谈 + PRD互评

🔍 用户访谈技巧

  • 问"最近一次"而不是"你总是"
  • 问场景，不问偏好
  • 注意"沉默"——没说的话可能更重要
  • 验证假设，不推销方案

👥 PRD互评

  • 小组互相审查PRD
  • 发现隐藏假设和偏见
  • AI分析PRD中的文化/性别/社会偏见
  • 类Apple Health案例重演

作业：完成2次用户访谈 + PRD初稿（含两个具体Persona）

Part 5 JTBD框架：砍需求的商业逻辑

核心转变：从"功能列表"到"Jobs to Be Done"

用户不是买产品，而是"雇佣"产品完成任务。每个需求必须包含三层：功能+情感+社会

📊 机会得分公式

机会得分 = 重要性 + (重要性 - 满意度)

高重要性 + 低满意度 = 未满足需求 = MVP核心

🎯 关键时刻 Moments of Truth

情绪最激烈、决定用户去留的互动节点。
如：第一次匹配成功、第一次收到防网暴提示。

MVP80%资源 → 打磨这些瞬间

Part 6 Prompt Engineering：PRD的AI工作流

🚫 禁止：一键生成PRD（得到废话）

✅正确：链式提示（Prompt Chaining）=任务分解 + 步步收敛

1️⃣

发散

输入人群+JTBD框架
让AI生成10个痛点

2️⃣

收敛

AI用机会得分打分
筛出前3个痛点

3️⃣

输出

Markdown格式输出
PRD范围层规格

SOMA评估法：让AI做恶人砍需求
写Prompt让AI对每个功能按"重要性/可行性/价值对齐度"打分，强制建议砍掉最低2个

W2 总结

  • ✅ AI助手（被动）vs AI Agent（主动）
  • ✅ 双钻模型：先找根本原因，再给答案
  • ✅ 痛点驱动需求，痒点靠边站
  • ✅ PRD = 给AI的食谱，Garrett战略/范围层
  • ✅ 两个具体Persona是PRD第一页
  • ✅ 用户访谈 + 小组互评发现隐藏偏见
  • ✅ JTBD框架：功能+情感+社会三层需求
  • ✅ 机会得分公式 + 关键时刻聚焦
  • ✅链式提示（Prompt Chaining）+ SOMA评估让AI砍需求
