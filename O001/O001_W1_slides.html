Part 0 什么是 LLM？

🤖 LLM =超级高级的自动补全

一个读了世界上所有书的AI。
你给半句话，它预测下一个最可能出现的词。
它不是"思考"——是无数次训练出来的预测。

2026年主流模型

  • GPT-5.5 / Claude Opus 4.7
  • Sonnet 4.6 / Gemini 3.1 Pro
  • DeepSeek-V3.2

⚠️ 幻觉 Hallucination

AI自信地编造听起来真实但不存在的信息。
这是所有LLM的共同特点，不是bug。
你必须始终核对事实。

Part 1 Vibe Coding：人人都能创造

核心理念：和AI对话着构建，边做边改，不要求提前规划。

你告诉AI你想要什么 → 它给你代码 → 你判断结果 → 循环往复

✓ 适合

  • 学习新领域
  • 测试想法
  • 快速做小工具
  • 零基础起点

✗ 不适合

  • 高可靠性系统（医疗/航空）
  • 需要严格架构的生产代码

Part 1 Prompt = 言语指导行动

🔑 认知科学原理（Vygotsky）

儿童在遇到困难时会自言自语（Egocentric Speech），用语言规划和组织下一步动作。

Prompting不是"给机器下指令"——而是你在进行外化的"自言自语"和"认知规划"

告诉学生：不要把写提示词当成写代码，而要当成用自然语言梳理自己逻辑的过程

AI = 符号中介（Mediating Sign） —不仅是外部工具，更改变了学生内部的思维结构

Part 3 自然语言生成第一个页面

核心心法：说清楚想要什么，AI才能做对

就像和实习生说话——你给方向，它帮你实现。
不是写代码，而是用自然语言梳理你的逻辑。

好Prompt示例

"帮我做一个母亲和女儿对话的聊天界面，左边是妈妈的消息，右边是女儿的消息，每条消息有头像、时间戳和已读状态。"

Part 4 AI安全与创造者责任

⚠️ AI能帮你创造，也能被坏人滥用

作为创造者，你有责任确保你的工作不会伤害他人。

RSA攻击策略

  • Role-assignment：诱导扮演特定角色
  • Scenario-pretexting：虚构场景绕过限制
  • Action-solicitation：诱导生成恶意代码

实践规则

  • 始终检查AI生成代码中的可疑API调用
  • 警惕数据收集行为
  • 不执行来源不明的代码

Part 5 AI的"默认偏见"与边缘案例

AI默认走向"大多数情况"

训练数据反映历史偏见和文化刻板印象。对于性别相关社会创新App，必须主动Prompt边缘案例。

📱 Apple Health案例（2014）

设计团队多样性不足，导致月经追踪器未纳入——直到2019年才修复。数百万女性健康数据需求被默认忽略。

练习：让AI分析你的PRD，问"有哪些边缘案例？有哪些用户群体可能被忽略？"

W1 总结

  • ✅ LLM = 超级自动补全，不是思考
  • ✅ Vibe Coding = 边做边改，不要求完美规划
  • ✅ Prompt = 言语指导行动，是元认知训练
  • ✅ AI = 符号中介，改变思维结构
  • ✅ 始终检查代码安全，核对AI事实
  • ✅ 主动问边缘案例，不让偏见漏网
